Slice 3 · Explanations & GapsHow it works

The problem it solvesGood scores, but two silences

"Why this person?"

Slice 2 gives strong bands, but a staffing manager still has to reverse-engineer the reasoning. Trust needs a stated why, grounded in real evidence.

"What if no one fits?"

For an unfillable role, a ranked list of weak matches — or worse, an empty screen — is useless. The manager needs to know what's missing and who's closest.

Slice 3 turns numbers into language a human can act on, and turns "no match" into a constructive answer.

Grounded explanations3 / 7

Explaining the orderWhy #3 isn't #1

For lower-ranked candidates, the system names the specific reason they sit below — never a vague "lower score".

This makes the advisory ranking legible: the manager sees the one lever (a missing skill, a later start, unverified feedback) and can decide whether it matters for their situation.

#1 — strong across skill & feedback
#3 — "below #1: adjacent skill only, not exact"
#5 — "below #3: no internal feedback yet"

Beyond the terminalA machine-readable answer

The same result is emitted as structured JSON — per-dimension bands, signals, confidence, and a snapshot timestamp.

This is what lets the engine plug into a future UI, an ATS, or a Slack bot — and it's how the Slice 4 eval harness reads results to score them automatically.
